🧠 BlackHat Knowledge Base

Web: SQL Injection

  • ' OR 1=1 --
  • UNION-based Injection
  • Time-based Blind SQL
  • SQLMap Automation
  • Prevention: Prepared Statements, ORM, Input Validation

Web: Cross-Site Scripting (XSS)

  • Reflected XSS
  • Stored XSS
  • DOM-based XSS
  • Payload: <img src="javascript:alert('XSS')">
  • Mitigation: Content Security Policy (CSP), HTML Encoding

System: Command Injection

  • Example: ping google.com; ls -la
  • Methods: Shell escape characters
  • Mitigation: Input validation, least privilege

System: Buffer Overflow

  • Overflow exploits memory corruption
  • Example: AAAA...AAAA
  • Exploit: EIP overwrite for arbitrary code execution
  • Mitigation: Stack canaries, ASLR, DEP

Social Engineering: Phishing Attack

  • Target: User credentials and sensitive data
  • Techniques: Email spoofing, fake websites
  • Mitigation: User awareness, multi-factor authentication

Network: Man-in-the-Middle (MITM)

  • Example: SSL stripping, ARP spoofing
  • Tools: Wireshark, mitmproxy
  • Mitigation: TLS, VPN, HTTPS

Web: Directory Traversal

  • Attack: Accessing restricted files via ../../etc/passwd
  • Prevention: Input sanitization, proper access controls
  • Tools: Burp Suite, OWASP ZAP

System: Remote Code Execution (RCE)

  • Vulnerability: Untrusted input execution
  • Example: WebShell, System backdoor
  • Mitigation: Input validation, code reviews

Web: Cross-Site Request Forgery (CSRF)

  • Attacks: Forced actions on behalf of an authenticated user
  • Example: Form submission manipulation
  • Mitigation: Anti-CSRF tokens, SameSite cookies

Web: Clickjacking

  • Technique: Framing the target site with transparent content
  • Mitigation: X-Frame-Options header, Content Security Policy (CSP)

Web: Session Fixation

  • Attack: Attacker sets session ID for victim
  • Mitigation: Regenerate session IDs on login, use secure cookies

Network: DNS Spoofing

  • Exploit: Redirecting a victim’s DNS queries to malicious IPs
  • Tools: Ettercap, DNSChef
  • Mitigation: DNSSEC, DNS-over-HTTPS

Network: ARP Spoofing

  • Attack: Poisoning the ARP cache to intercept traffic
  • Tools: Ettercap, Cain & Abel
  • Mitigation: Static ARP entries, use of VPN

Network: Evil Twin Attack

  • Technique: Fake Wi-Fi access point to capture credentials
  • Mitigation: Use of VPN, strong WPA2 passwords

General: Zero-Day Vulnerability

  • Definition: A vulnerability that is unknown to the vendor
  • Exploited before a patch is released
  • Mitigation: Prompt patching, monitoring CVE databases

Malware: Trojan Horse

  • Example: Malware disguised as legitimate software
  • Mitigation: Antivirus software, email filtering
